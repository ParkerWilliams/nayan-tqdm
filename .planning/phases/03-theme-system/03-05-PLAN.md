In `__init__` body, after terminal detection and fallback decision, add theme resolution:

```python
# Theme resolution
self._animation = None
if not self.disable:
    tier = resolve_render_tier(self._terminal, render_level)
    theme_name = theme or get_default_theme()
    try:
        self._animation = get_theme(theme_name, tier)
    except Exception:
        # Absolute safety net -- if theme loading fails, fall back to no animation
        self._animation = None
```

This replaces the existing `self._animation = None` line. The animation thread startup logic in `_start_animation_thread()` already checks `self._animation is not None` and `self._use_fallback`, so no changes needed there.

**Important:** The `import nyanbar.themes` must be at module level in core.py to ensure all built-in themes are registered before any NyanBar instance tries to resolve a theme. This is safe because themes only register factories (no heavy work).
