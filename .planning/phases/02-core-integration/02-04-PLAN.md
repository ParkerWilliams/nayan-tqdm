**GREEN phase -- update `nyanbar/core.py`:**

Add the following to the existing NyanBar class:

1. **Animation state in `__init__`:**
   - `self._animation: Animation | None = None` -- set by theme system in Phase 3; for now, internal attribute
   - `self._anim_thread: threading.Thread | None = None`
   - `self._running = False`
   - Do NOT start animation thread in __init__. Start it lazily in `__iter__` or first `update()`.

2. **`_start_animation_thread()` method:**
   ```python
   def _start_animation_thread(self) -> None:
       if self._use_fallback or self.disable or self._animation is None:
           return
       if self._running:
           return  # already started
       self._running = True
       t = threading.Thread(target=self._animation_loop, daemon=True)
       t.start()
       self._anim_thread = t
   ```

3. **`_animation_loop()` method:**
   ```python
   def _animation_loop(self) -> None:
       interval = 1.0 / (self._animation.fps if self._animation else 12.0)
       while self._running:
           time.sleep(interval)
           with self._lock:
               if not self._running or self._closed:
                   break
               self._render_animation_frame()
   ```
   Key: lock is held ONLY during the render call, NOT during sleep.

4. **`_render_animation_frame()` method** (called from daemon thread, lock already held):
   - Compute elapsed = `time.monotonic() - self._start_t`
   - Compute progress = `self.n / self.total` if total else 0.0
   - Get animation lines from `render_animation(self._animation, progress, anim_width, elapsed)`
   - Compute stats string from `format_meter(...)`
   - Compose: animation lines with stats on the last line (stats appended to right of last animation line)
   - Render to terminal using `render_frame(composed_lines, self._prev_height, self.file)` from renderer.py
   - Update `self._prev_height`

5. **`_stop_animation_thread()` method:**
   ```python
   def _stop_animation_thread(self) -> None:
       self._running = False
       # Don't join -- daemon thread dies naturally
   ```

6. **Update `__iter__` to start animation thread:**
   - Call `self._start_animation_thread()` at the beginning of __iter__ (after disable check)
   - The two-gate throttle in __iter__ controls STATS refresh, not animation refresh (animation is on its own timer via daemon thread)

7. **Update `close()` for animation cleanup:**
   ```python
   def close(self) -> None:
       if self._closed:
           return
       self._closed = True
       self._stop_animation_thread()
       if self.disable:
           return
       with self._lock:
           if self.leave:
               self._render_final()  # final bar state
               self.file.write("\n")
           else:
               from .renderer import erase_lines
               erase_lines(self.file, self._prev_height)
           self.file.flush()
   ```

8. **Update `_refresh()` for two-gate throttle integration:**
   - The _refresh method (called from main thread after gates pass) should:
     - Update EMA with (dn, dt)
     - If using fallback: render fallback bar
     - If using animation: just update stats state (daemon thread handles rendering)
     - Auto-tune miniters

9. **`_render_final()` method** (for leave=True):
   - If animation: render completion state (100% progress)
   - If fallback: render final fallback bar
   - Both followed by newline

10. **maxinterval gate**: In `update()`, after the mininterval check, also check: if `time.monotonic() - self._last_print_t >= self.maxinterval`, force a refresh regardless of miniters. This catches stalls.
