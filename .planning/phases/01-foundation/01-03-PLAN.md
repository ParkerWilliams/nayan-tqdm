<feature>
  <name>Frame and Animation Data Model</name>
  <files>nyanbar/models.py, tests/test_models.py</files>
  <behavior>
    AnimationMode enum:
    - WALK: sprite moves across bar with progress
    - CYCLE: sprite stays in place, frames cycle over time
    - FILL: animation pattern fills bar area

    Frame frozen dataclass:
    - lines: tuple[str, ...] (each line of the sprite)
    - display_width: int (computed in __post_init__ as max disp_len of lines)
    - height: int (computed in __post_init__ as len(lines))
    - Construction: Frame(lines=("=^.^=", " || ")) -> display_width=5, height=2
    - Frozen: frame.lines = ("new",) raises FrozenInstanceError
    - Empty lines tuple: display_width=0, height=0
    - Lines with ANSI codes: display_width computed on stripped text
    - Lines with CJK: display_width reflects double-width correctly

    Animation frozen dataclass:
    - name: str (theme name identifier)
    - frames: tuple[Frame, ...] (sequence of frames for animation)
    - fps: float = 12.0 (frames per second, configurable per theme)
    - mode: AnimationMode = AnimationMode.WALK (default mode)
    - completion_frame: Frame | None = None (optional frame for 100%)
    - frame_count: int (computed in __post_init__ as len(frames))
    - Construction: Animation(name="cat", frames=(frame1, frame2), fps=10)
    - Frozen: anim.fps = 24 raises FrozenInstanceError
    - frame_count auto-computed from frames tuple length
    - Single frame animation: frame_count=1, still valid

    Test cases:
    - Frame construction with basic ASCII lines
    - Frame display_width with mixed ASCII/CJK
    - Frame immutability (FrozenInstanceError)
    - Animation construction with defaults
    - Animation construction with custom fps
    - Animation construction with completion_frame
    - Animation immutability (FrozenInstanceError)
    - AnimationMode enum values
    - Edge: empty frames tuple
    - Edge: single-frame animation
  </behavior>
  <implementation>
    Use @dataclass(frozen=True, slots=True) for both Frame and Animation.
    Use object.__setattr__ in __post_init__ for computed fields (display_width, height, frame_count).
    Import disp_len from .utils for width calculation.
    Use enum.Enum for AnimationMode.
    frames and lines stored as tuples (not lists) for true immutability.
    Type annotations throughout with from __future__ import annotations.
    Note: models.py imports from utils.py, but utils.py has no imports from models.py — no circular dependency.
    The import means models depends on utils at runtime, but for Wave 1 parallel execution,
    Plan 01 (utils) should complete before Plan 03 tests run. However, the dataclass
    definition itself doesn't need utils — only __post_init__ does. Tests will need utils
    to be present. Mark this as a soft dependency — plans can still run in Wave 1 since
    the file can be created independently, but test execution order matters.
  </implementation>
</feature>
