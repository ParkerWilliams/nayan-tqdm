<feature>
  <name>Animation Engine</name>
  <files>nyanbar/engine.py, tests/test_engine.py</files>
  <behavior>
    select_frame_index(frame_count: int, elapsed: float, fps: float) -> int:
    - frame_index = int(elapsed * fps) % frame_count
    - elapsed=0.0, fps=12, 4 frames -> index 0
    - elapsed=0.1, fps=12, 4 frames -> int(1.2) % 4 = 1
    - elapsed=0.5, fps=12, 4 frames -> int(6) % 4 = 2
    - elapsed=1.0, fps=12, 4 frames -> int(12) % 4 = 0 (wraps)
    - Single frame (frame_count=1): always returns 0

    walk_position(progress: float, width: int, sprite_width: int) -> int:
    - position = int(progress * (width - sprite_width))
    - progress=0.0, width=80, sprite_width=5 -> 0
    - progress=0.5, width=80, sprite_width=5 -> 37
    - progress=1.0, width=80, sprite_width=5 -> 75
    - Clamped: progress < 0 -> 0, progress > 1.0 -> width - sprite_width
    - Edge: sprite_width >= width -> 0 (no room to move)

    cycle_position(width: int, sprite_width: int) -> int:
    - Returns fixed center position: (width - sprite_width) // 2
    - Clamped to >= 0

    fill_pattern(frame: Frame, width: int) -> list[str]:
    - For each line in frame.lines, repeat/tile the line content to fill `width` columns
    - Uses disp_len for width-aware tiling (CJK chars take 2 columns)
    - Truncates cleanly at width boundary

    render_animation(
        animation: Animation,
        progress: float,
        width: int,
        elapsed: float,
    ) -> list[str]:
    - If progress >= 1.0 and animation.completion_frame is not None:
      return completion_frame positioned at rightmost position
    - Select frame via select_frame_index(animation.frame_count, elapsed, animation.fps)
    - Based on animation.mode:
      - WALK: position = walk_position(progress, width, frame.display_width)
             Build lines with sprite at position (pad left, sprite, pad right)
      - CYCLE: position = cycle_position(width, frame.display_width)
              Build lines with sprite at fixed position
      - FILL: return fill_pattern(frame, width)
    - Return list of output lines (one per frame line)

    Test cases:
    - Frame selection at various elapsed times (mock time not needed — elapsed is a parameter)
    - Frame selection wraps correctly
    - Walk position at 0%, 50%, 100%
    - Walk position with negative progress (clamp to 0)
    - Walk position with progress > 1 (clamp to max)
    - Cycle position centered
    - Fill pattern basic tiling
    - render_animation in WALK mode produces correct padded lines
    - render_animation in CYCLE mode keeps sprite centered
    - render_animation in FILL mode tiles across width
    - render_animation at 100% with completion_frame uses it
    - render_animation at 100% without completion_frame uses normal frame
    - render_animation with single-frame animation
  </behavior>
  <implementation>
    All functions are pure — take data in, return data out. No side effects, no I/O.
    select_frame_index takes elapsed as float parameter (caller provides time.monotonic() - start_time).
    walk_position uses simple linear mapping: int(progress * available_space).
    Padding in render_animation uses space characters for left/right fill.
    For WALK mode output lines: " " * position + frame_line + " " * remaining
    Each output line should be exactly `width` display columns (pad to fill).
    Import Animation, Frame, AnimationMode from .models.
    Import disp_len from .utils for width-aware operations.
    Tests create simple Frame/Animation fixtures for testing.
  </implementation>
</feature>
